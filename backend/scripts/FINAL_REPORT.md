### 📋 系统特性

#### ✅ 严格真实性约束
- **禁止生成示例数据** - 系统不会创建任何虚假数据
- **禁止自动补全** - 不推测或补全缺失数据
- **只从真实源获取** - 官方附件、JSON API、HTML表格
- **明确报错机制** - 无法获取时返回 `FAILED_TO_FETCH_REAL_DATA`

#### ✅ 数据来源优先级
1. **广东省教育考试院官方附件**（Excel/PDF）- 最高优先级
2. 官方JSON接口（如有）
3. 高考直通车
4. 掌上高考
5. 中国教育在线

#### ✅ 数据验证逻辑
- 官方数据存在 → `verified = true`
- 2个以上第三方来源一致 → `verified = true`
- 只有1个来源 → `verified = false`
- 无任何来源 → 记录到 `missing_report`

#### ✅ 数据完整性要求
- **最少3000条**专业组记录
- **必须包含**：民办院校、独立学院、中外合作专业组、公办本科、985/211
- **必需字段**：院校名称、专业组代码、最低分、最低排位

#### ✅ 专业组展开
- 从阳光高考网或高校官网采集专业组映射
- 创建 `mapping.json` 存储映射关系
- 无法映射的标记 `has_major_details = false`

#### ✅ 数据清理
自动删除包含以下特征的记录：
- 院校名称包含"一般"、"测试"、"模拟"
- 最低排位为0或空
- 最低分 < 100 或 > 750

### 📁 输出文件

1. **guangdong_2025_data_YYYYMMDD_HHMMSS.csv** - 主数据文件
