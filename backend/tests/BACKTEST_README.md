# 推荐系统回溯测试工具使用说明

## 📋 概述

这是一个用于验证推荐系统准确性和可靠性的回溯测试工具。通过历史录取数据来测试推荐系统的命中率和准确性。

## 🎯 功能特点

- ✅ **分层随机抽样**: 按位次段分层抽取测试样本，确保测试代表性
- ✅ **自动化测试**: 自动调用推荐接口，批量测试500条样本
- ✅ **多维度分析**: 整体命中率、分位次段表现、概率准确度、性能指标
- ✅ **详细报告**: JSON、CSV、HTML多种格式的测试报告
- ✅ **未命中分析**: 详细分析未命中案例，帮助优化算法
- ✅ **进度显示**: 实时显示测试进度和当前命中率
- ✅ **错误处理**: 完善的错误处理和日志记录

## 📊 测试原理

### 1. 测试数据来源
- **数据文件**: `../data/major_rank_data.json` (82.9万条记录)
- **筛选条件**: 广东省 + 2025年数据
- **分层抽样**: 按位次段分为6个层次，每层按比例抽样

### 2. 命中判定规则
- **院校命中**: 被录取的院校出现在推荐列表中（任何专业）
- **精确命中**: 被录取的院校+专业完全匹配
- **概率准确度**: 预测概率与实际命中率的对比

### 3. 统计指标
- **整体指标**: 总命中率、精确命中率
- **分位次段指标**: 各分数段的命中率表现
- **概率准确度**: 按预测概率区间的实际命中率
- **性能指标**: 响应时间、成功率等

## 🚀 使用方法

### 方法1: 使用批处理脚本（推荐）

双击运行 `run_backtest.bat`，按照提示选择测试模式：

```
1. 快速测试 (50条样本, ~1分钟)
2. 完整测试 (500条样本, ~10分钟)
3. 自定义样本量
4. 退出
```

### 方法2: 命令行直接运行

```bash
# 进入测试目录
cd backend/tests

# 完整测试 (500条)
python backtest.py

# 快速测试 (50条)
python backtest.py --quick

# 自定义样本量 (200条)
python backtest.py --sample 200

# 重新运行测试
python backtest.py --rerun
```

## 📈 输出文件

测试完成后，会在 `results/` 目录生成以下文件：

### 1. JSON详细报告
`backtest_report_20250109_143022.json`
- 包含所有统计数据
- 分位次段详细指标
- 概率准确度分析
- 性能指标

### 2. 未命中案例详情
`backtest_unhit_details_20250109_143022.json`
- 前50条未命中案例
- 每条包含实际录取结果和推荐结果
- 用于分析失败原因

### 3. CSV统计表
`backtest_hit_rate_by_rank_20250109_143022.csv`
- 按位次段分组的命中率数据
- 可用于Excel分析和图表绘制

### 4. HTML可视化报告
`backtest_report_20250109_143022.html`
- 美观的可视化报告
- 图表化展示各项指标
- 可直接在浏览器中查看

## 🔧 技术细节

### 依赖包
- `requests`: HTTP请求库
- `tqdm`: 进度条显示

### API接口
- **端点**: `POST http://localhost:8000/api/v1/recommend`
- **参数**:
  ```json
  {
    "rank": 15000,
    "province": "广东",
    "subjects": ["物理", "化学", "生物"],
    "preference": "balanced"
  }
  ```

### 数据流程
1. 加载录取数据 (82.9万条)
2. 筛选广东省+2025年数据
3. 按位次段分层随机抽样
4. 批量调用推荐API
5. 统计分析命中情况
6. 生成测试报告

## ⚠️ 注意事项

1. **后端服务**: 测试前请确保后端服务已启动 (`python -m app.main`)
2. **数据文件**: 确保 `../data/major_rank_data.json` 文件存在
3. **网络连接**: 推荐接口需要能访问 `http://localhost:8000`
4. **测试时间**: 完整测试约需10分钟，请耐心等待
5. **样本量**: 如果数据不足，脚本会自动调整样本量

## 🐛 常见问题

### Q1: 提示"无法连接到API"
**A**: 请确保后端服务已启动：
```bash
cd backend
python -m app.main
```

### Q2: 数据文件不存在
**A**: 请确保数据文件路径正确：
```bash
backend/data/major_rank_data.json
```

### Q3: 样本量不足
**A**: 脚本会自动调整样本量，或您可以手动指定较小的样本量：
```bash
python backtest.py --sample 100
```

### Q4: 测试过程中断
**A**: 检查后端服务是否正常运行，可以重新运行测试：
```bash
python backtest.py --rerun
```
