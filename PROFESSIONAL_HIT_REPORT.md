## 🔍 问题分析

### 1. 专业组命中率低的原因

#### 数据结构问题
- **专业组代码分散**: 现有数据中专业组代码分布不统一
- **映射关系不足**: 634个实际专业组，但映射表可能覆盖不完整
- **专业数量限制**: 每个专业组平均只有2.9个专业，映射关系薄弱

#### 算法策略问题
```python
# 当前算法优先级
1. 同院校同专业组推荐 (优先级最高)
2. 同院校其他专业推荐
3. 其他院校推荐
```

**问题**: 算法优先推荐同院校，但专业组匹配权重不够

### 2. 精确专业命中率分析

**16.70%的精确命中率** 说明：
- 推荐算法能较好地匹配同院校
- 但专业名称完全匹配的要求过严格
- 实际录取中，学生常被录取到同院校的其他专业

### 3. 院校推荐表现优秀

**73.84%的院校命中率** 说明：
- 算法在院校层面的推荐非常准确
- 冲稳保垫的院校选择范围合理
- 位次匹配策略有效

### 方案二：调整推荐算法权重

```python
def calculate_recommendation_score(record, test_case):
    """计算推荐得分（调整权重）"""
    score = 0

    # 院校匹配（权重：50分）
    if record["university"] == test_case["university"]:
        score += 50

    # 专业大类匹配（权重：30分）
    record_major_category = get_professional_category(record["major"])
    target_major_category = get_professional_category(test_case["major"])
    if record_major_category == target_major_category:
        score += 30
    elif is_similar_category(record_major_category, target_major_category):
        score += 15  # 相似类别给一半分数

    # 位次接近度（权重：20分）
    rank_diff = abs(record["min_rank"] - test_case["min_rank"])
    score += max(0, 20 - rank_diff / 5000)

    return score
```
